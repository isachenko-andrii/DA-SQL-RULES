У цьому розділі ми будемо розглядати лише частину мови SQL, а саме DQL, бо саме вона використовується аналітиками даних для отримання потрібної інформації в необхідному вигляді.

  **DQL (Data Query Language) — це категорія команд у мові SQL, призначена виключно для пошуку, вибірки та отримання інформації з баз даних.**  

Для аналітика знання SQL (DQL) є таким же базовим навиком, як для водія вміння крутити кермо. Це головний інструмент для отримання, фільтрації та обробки сирих даних безпосередньо в сховищі без завантаження мільйонів рядків на власний комп'ютер.  

  Основні завдання, які дата-аналітик вирішує за допомогою SQL:  

 - **Збір (вибірка) даних:** написання запитів (наприклад, SELECT) для отримання точної інформації про клієнтів, продажі чи транзакції
 - **Фільтрація та сортування:** відсіювання непотрібних даних за певними умовами (блоки WHERE, ORDER BY)
 - **Об'єднання таблиць:** зведення даних із різних джерел для отримання повної картини (оператори JOIN)
 - **Групування та агрегація:** підрахунок сум, середніх значень, максимумів або мінімумів (наприклад, GROUP BY, SUM, AVG)  
 - **Аналітика в часі та ранжування:** використання віконних функцій для розрахунку змін, рейтингів та ковзних середніх.  

##  Базовий синтаксис DQL 
---  

  Базовий синтаксис DQL (Data Query Language) обертається навколо однієї головної команди — SELECT, яка використовується для вибірки даних. Основа конструкції виглядає наступним чином:  
  
<code>**sql** 
**SELECT** стовпець_1, стовпець_2   
**FROM** таблиця   
**WHERE** умова;  
</code>   
  
Основні компоненти запиту:  
  
**SELECT:** визначає, які саме стовпці ви хочете отримати. Використання зірочки (*) обирає всі стовпці таблиці.  
**FROM:** вказує таблицю, з якої витягуються дані.  
**WHERE:** фільтрує дані, повертаючи лише ті рядки, що відповідають певній умові.  
  
Поширені розширення базового синтаксису:  
  
**ORDER BY:** сортує результат за певним стовпцем (за зростанням ASC або за спаданням DESC).  
**LIMIT / OFFSET:** обмежує кількість рядків, що повертаються, та пропускає певну їх кількість (корисно для пагінації).  
**GROUP BY:** групує рядки із однаковими значеннями для використання агрегатних функцій (наприклад, SUM(), AVG(), COUNT()).  
**HAVING:** фільтрує результати після застосування GROUP BY.  
  
## Правило опису шляху звернення до таблиці  
---  
  
 Шлях до таблиці в SQL будується за принципом ієрархії:  
 
 **Сервер.База_Даних.Схема.Таблиця**  
   
 Повний шлях використовується рідко — зазвичай вказують лише необхідний мінімум залежно від того, наскільки складні зв'язки між базами.  

**Стандартний формат шляху**  
  
Повна структура повного шляху виглядає так:  
  
<code>[Linked_Server].[Database_Name].[Schema_Name].[Table_Name]</code>  
  
де:  
  
**Linked Server (Сервер):** віддалений сервер БД (за потреби).  
**Database Name (База даних):** ім'я конкретної бази даних (каталогу).  
**Schema Name (Схема):** логічна група об'єктів усередині БД (наприклад, dbo, public).  
**Table Name (Таблиця):** назва самої таблиці.  
  
**Рівні доступу до таблиці (приклади)**  
  
Вибір формату залежить від вашої поточної сесії та конфігурації сервера:  

**1. Тільки назва таблиці (Найчастіший варіант)**  

Використовується, якщо ви вже підключені до потрібної бази даних, а таблиця знаходиться у схемі за замовчуванням (у більшості СУБД це dbo або public).
  
<code>**sql**  
SELECT * FROM users;</code>  

**2. Назва схеми + таблиця**  

Рекомендовано для уникнення плутанини, якщо у базі даних є кілька схем (наприклад, sales або hr), або як гарна практика написання коду.
  
<code>**sql**    
SELECT * FROM hr.employees;</code>  
  
**3. База даних + схема + таблиця**  
  
Використовується, якщо потрібно зробити запит до однієї бази даних, перебуваючи всередині іншої, або для явного доступу.  
  
<code>**sql**  
SELECT * FROM shop_db.dbo.orders;</code>  
  
**4. Повний шлях (Сервер + База даних + Схема + Таблиця)**  
  
Застосовується у корпоративних системах для крос-серверних запитів або для доступу до таблиць на віддалених серверах (Linked Servers).  
  
<code>**sql**  
SELECT * FROM mssql_server_01.sales_db.sales.transactions;</code>
